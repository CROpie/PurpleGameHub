----- basic setup -----

/home/chris/nginxtest/index.html to be served @ localhost:1111

// create new site config
sudo touch /etc/nginx/sites-available/nginxtest
sudo nano /etc/nginx/sites-available/nginxtest

server {
    listen 1111;
    server_name localhost;

    root /home/chris/nginxtest;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}

// run this command to create sybobolic link in sites-enabled
// sites-available: all possible site configs
// sites-enabled: just the ones that nginx should load
sudo ln -s /etc/nginx/sites-available/nginxtest /etc/nginx/sites-enabled/

A change in sites-available is immediately reflected in sites-enabled

Nginx’s main config (/etc/nginx/nginx.conf) typically has a line like:
include /etc/nginx/sites-enabled/*; (load every config in sites-enabled)

// to disabled loading a config, just delete it from sites-enabled

// restart nginx
sudo nginx -t
sudo systemctl reload nginx

localhost:1111: Success!

// if not success, this might help:
sudo tail -f /var/log/nginx/error.log

        try_files $uri $uri/ =404;

This is saying:
root is /home/chris/nginxtest
index should be located here: /home/chris/nginxtest/hello/index.html

O correct

server {
    listen 1111;
    server_name localhost;

    location /hello {
        alias /home/chris/nginxtest/;
        index index.html;
    }
}

// If you use alias inside a location block, you don’t inherit the root from the server block.
// /hello will be replaced completely by alias: /home/chris/nginxtest/
